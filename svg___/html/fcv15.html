Verificando respuestas de remotos

La respuesta a un mensaje invite contiene entre otros campos, el código de estatus según el estándar HTTP. Los códigos de respuesta soportados para un mensaje invite son:

Código Descripción
200,201 CREATED, OK: Indica que el endpoint puede contactar al remoto, que determinó una ruta de salida adecuada y que se creó satisfactoriamente la relación para enrutar los mensajes recibidos.
202 ACCEPTED: Indica que se podrán enviar mensajes hacia el remoto, pero no se podrán recibir, es decir, solo salida.
302 TEMPORARY REDIRECT: El remoto no pudo ser contactado por el endpoint, ofrece enrutamiento donde quizá se pueda contactar.
303 SEE OTHER: El remoto no puede ser contactado por el endpoint, se ofrece una nueva forma para contactarlo.
400 BAD REQUEST: La petición contiene un JSON mal formado o con parámetros insuficientes.
404 NOT FOUND: El endpoint no puede contactar al remoto y no conoce una forma de hacerlo.
500 INTERNAL ERROR: El endpoint presentó un error al tratar de seguir el flujo para dar una respuesta al invite, el switch puede decidir reintentar.
