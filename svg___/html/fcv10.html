Verificando token detonante

Cuando el switch encuentra la IR, compara el token que obtuvo como parámetro con el token en la Inbound Rule, si coinciden significa que se recibió un token que detona una conversación. CBOTESTAFETAV1 para nuestro ejemplo, si es el caso, continúa al proceso para crear una nueva conversación, en caso contrario, se trata de un mensaje para una conversación existente, el switch identifica a que conversación pertenece el mensaje y lo inyecta al flujo de esa conversación.
