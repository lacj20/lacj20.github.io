Terminando la conversación

Para terminar la conversación, el switch envía mensajes SWITCHBYE a todos los participantes, este tipo de mensaje le informa a los endpoints que la conversación ha finalizado, contiene datos como el id de la conversación y la URI del remotó que terminó su participación. La respuesta esperada para este mensaje es un objeto que contiene el statusCode según el estandar HTTP. Los código de respuesta soportados son:

Código Descripción
200 OK: Indica que los recursos ocupados fueron liberados correctamente.
400 BAD REQUEST: La petición trae un JSON malformado o con parámetros insuficientes.
404 NOT FOUND: El endpoint no conoce el URI del remoto.
500 INTERNAL ERROR: El endpoint presentó un error al tratar de seguir el flujo de proceso de un BYE. Es decisión del switch reintentar la solicitud o no.
